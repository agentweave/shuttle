Five skills, one workflow.

Scope work, add tasks, start the heartbeat, check progress, stop when done.

/shuttle:kickoff
Interactive scoping session. Asks questions, decomposes your work into well-sized tasks, writes the task file.
/shuttle:add
Quick-add a single task. Creates the task file if it doesn't exist. Good for one-offs.
/shuttle:start
Start the heartbeat. Installs the idle suppression hook. Configurable interval and task file path.
/shuttle:status
Show task counts by status and heartbeat state. Quick progress check without interrupting work.
/shuttle:stop
Full cleanup. Removes the cron, hook, and session hints. Task file is preserved.

How it works

A task file, a heartbeat, and a shell pre-check. Three layers, zero wasted tokens.

01

Task file

Plain markdown with one rule: **Status:** ready | in-progress | done. That's the whole protocol.

02

Heartbeat

CronCreate fires every 15 minutes. The model reads the task file, picks up work, updates progress.

/shuttle:start 5m
03

Idle suppression

A shell pre-check greps for active tasks before invoking the model. No tasks? Tick skipped. Zero tokens.

Quick start

Two paths to get going.

# Guided scoping session
/shuttle:kickoff
/shuttle:start
# Or add tasks directly
/shuttle:add "Fix the login bug"
/shuttle:add "Add rate limiting"
/shuttle:start
# Check progress anytime
/shuttle:status
